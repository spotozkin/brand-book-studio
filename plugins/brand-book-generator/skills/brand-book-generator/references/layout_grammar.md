# Layout Grammar — Composing a Unique Book Every Time

`assets/brand_book_template.html` is a **component quarry, not a document**. Never ship
its assembly order, its furniture copy, or its component counts as-is. Every book is a
bespoke HTML assembly: pick one option per axis below (or invent within the axis), driven
by the posture (`document_postures.md`) and the brand's own character. Two books built by
this skill should never be identifiable as siblings.

The test before export: put this book next to the LEGO and Rolex reference outputs (the
known-cloned pair). If a stranger could sort it into their family by skeleton alone —
same cover formula, same chapter names, same three-stat bar, same sign-off pattern — it
fails. See `anti_fingerprint_checklist.md` for the mechanical version of this test.

## Axis 1 — Cover archetype (pick one, vary the details)

- **Split panel** — two-color vertical/horizontal split. (This is the quarry template's
  default and appears in both reference outputs — use it RARELY and only when two-color
  tension is genuinely the brand's story.)
- **Full-field** — one brand color edge to edge, mark or wordmark alone, tiny metadata.
  Restrained brands, spec books.
- **Typographic cover** — the brand name or tagline set enormous, cropped or bleeding
  off-page; no blocks at all. Voice-led brands.
- **Contained/bordered** — content inside a device frame (box, border, keyline) when the
  brand has a container device.
- **Pattern-field** — the brand motif tiled as the entire cover surface, title knocked
  out. Pattern-strong brands.
- **Manifesto cover** — opens with a full paragraph of voice, brand name subordinate.
  Mission-led brands.
- **Utility cover** — looks like a technical document: document number, version, date,
  distribution tier. Spec and rebrand books. Include real document-control furniture
  (version, date, owner) — not decoration, an actual register.

## Axis 2 — Contents treatment

Options: classic numbered TOC · TOC as full-page typographic index · marginal/sidebar
contents · chapter-opener strip (no TOC page; each chapter self-announces) · NONE
(one-sheets, short books ≤8pp — a TOC in a 7-page book is padding). Never reuse the
literal heading "What's inside." — the TOC heading, like all furniture copy, is written
fresh in the brand's voice or left as a plain functional label ("Contents").

## Axis 3 — Heading register (applies to every heading in the book)

Pick ONE register and hold it consistently; the register is a brand decision:
- **Neutral/functional** — "Color", "Typography", "Logo". Spec books, restrained brands.
- **In-voice** — headings written the way the brand talks (a playful brand gets playful
  headings; a reverent brand gets reverent ones). The default for narrative and
  voice-led books. Written fresh per brand — never reused across books.
- **Aphorism heads** — each section opens with a short witty/thematic line above the
  functional label. Personality brands.
- **FAQ register** — sections framed as questions the reader would actually ask ("What
  color is our blue, exactly?"). Approachable, internal-facing books.
- **Annotation layer** — neutral headings plus a running margin voice that comments on
  the rules. Editorial brands; hardest to execute, highest character.

## Axis 4 — Divider strategy

Full-bleed color dividers (the quarry default, used twice at fixed positions in the
cloned outputs) are ONE option: also consider photographic/pattern dividers · thin rule +
big numeral chapter openers · no dividers at all (continuous flow with strong chapter
openers) · divider only at one major hinge (strategy→system). Vary the COUNT and the
POSITIONS, not just the color.

## Axis 5 — Voice section format

The cloned outputs both used: 3 trait cards → we-are/we-are-not → 3 feeling cards →
say-this/not-this. That is one format. Alternatives:
- **Before/after/because rewrites** — real off-brand copy rewritten on-brand, with a
  one-line "because" naming the rule applied. High teaching value.
- **Attribute → mechanical rule tree** — each tone adjective decomposed into concrete,
  checkable writing rules (sentence length, person, punctuation bans), with a recap table.
- **Vocabulary register** — words we use / words we never use, with the reasoning.
- **In-voice demonstration + codification** — the section itself written in the brand
  voice, with margin annotations naming each rule as it's used. (Demonstration must
  ALWAYS be paired with codification — never wit without rules.)
- **Compact trait table** — for spec books and one-sheets: adjective, sounds-like,
  never, one line each.
Vary the NUMBER of traits (2-5) and feelings — three-of-everything is a fingerprint.

## Axis 6 — Do/don't format (the corpus documents 18+ distinct formats; use variety)

Prose paragraph of prohibitions · illustrated misuse grid with slash marks · paired
correct/incorrect columns (always include a POSITIVE wing — books that only say "don't"
teach less) · bounded-permission phrasing ("you may X only when Y") · severity-tiered
register (never / avoid / prefer) · self-interrogating FAQ ("Can I put the logo on a
photo? Only if…") · one-line bans embedded per-rule rather than collected. Pick what the
posture needs: spec books get registers; one-sheets get 2-3 hard bans.

## Axis 7 — Type specimen format

Specimen ladder (Display→H1→H2→Eyebrow→Body — the quarry default) · **specimen-by-role**
(each specimen IS a rule: the H1 sample states the H1 rule in itself) · paragraph-level
live specimen (a real passage set in the system, annotated) · character-set showcase
(A-Z, numerals, punctuation — verify the alphabet is complete if you set one) ·
conditional table (size band / environment → face, per the environment-conditional type
rule). Always include: license source, fallback stack for web contexts.

## Axis 8 — Mark chapter format

Vary: variation-grid size and contents (only show variants that actually exist — never
fabricate a reversed version from a single supplied file) · clearspace expressed in a
letterform unit of the mark itself (an "O", a stud, a stroke width) rather than abstract
padding, whenever the mark offers a natural unit · plain-language mark description in
prose (what IS this mark — always include; it survives when images break) · mark-anatomy
split (name the parts, note fragile ones) · dual-tier tolerances (minimum AND preferred)
for spec books · per-asset chapters when there are multiple marks (one chapter per mark,
single-source cross-references — never re-document a shared mark twice).

## Axis 9 — Applications format

Touchpoint spec cards + stacked web-hero composite (the quarry default; both cloned
outputs used it, including an identical three-stat credibility bar) is one option. Also:
logo×touchpoint assignment matrix (which mark variant goes where — spec books) ·
full-page single-application showcases · annotated real-artifact mockups (a business
card, a social tile, an email header, drawn at realistic proportion) · before/after
migration pairs (rebrand books) · none (one-sheets). If a stats/credibility bar appears,
vary its existence, count, and form — three stats in a colored band is now a fingerprint.

## Axis 10 — Back cover / closer

Two-word sign-off + metadata footer (both cloned outputs) is retired as a default.
Options: tagline restated alone · pattern-field with mark · document-control block
(version, owner, next review date — spec/rebrand) · a final in-voice line · contact/
escalation block as the closer · simple mirror of the cover. Whatever closes the book,
the escalation owner must appear SOMEWHERE in the document.

## Component-count variation (quiet fingerprints)

Counts are tells. Vary deliberately: number of tone traits (2-5), feelings (0-4), proof
points (2-5), stats in any bar (0-5), logo-variation tiles (3-8, only real ones),
touchpoint examples (1-6), palette swatches per row. If every generated book has three
of everything, the skill is showing.

## CSS/structure guidance

The quarry template's CSS classes (`.page`, `.section-label`, `.tone-grid`, etc.) are
reusable plumbing — reuse the plumbing, not the floor plan. For each book: write a fresh
`<body>` assembly; restyle proportions (margins, column counts, heading scale ratios,
eyebrow treatment) to the brand rather than inheriting the quarry's editorial defaults;
keep `@page { size: 8.5in 11in; margin: 0; }` and the WeasyPrint pipeline exactly as
documented in SKILL.md Step 5 — the RENDERING stack is fixed; the DESIGN is not.
